# Problem Statement

## Context
Potato aggregators and cooperative collection points in Musanze, Rwanda
currently manage orders using paper slips, WhatsApp messages, and mental math.
This causes real financial losses every market day.

## Core Problems
1. Wrong totals — quantity × price is calculated by hand and often wrong
2. No receipts — disputes happen at pickup because nothing is written properly
3. Lost records — paper gets wet, lost, or is never written at all
4. No overview — aggregators don't know how much they bought today or from whom

## Proposed Solution
A simple web app that lets an aggregator (admin) log in, register suppliers,
create orders with auto-calculated totals, print receipts, and view a daily dashboard.

## Who Benefits
- **Aggregators / admins** — accurate records, daily totals, printable receipts
- **Suppliers / farmers** — fair transactions, proof of delivery
- **Cooperative managers** — oversight of daily activity
